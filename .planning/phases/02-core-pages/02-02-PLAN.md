**Step 1: Create src/actions/index.ts** -- Astro Action for contact form, following the exact research Pattern 1:
- Import `defineAction`, `ActionError` from `astro:actions`, `z` from `astro/zod`, `Resend` from `resend`
- Instantiate Resend with `import.meta.env.RESEND_API_KEY`
- Define `contact` action with `accept: 'form'` and Zod schema:
  - `name`: z.string().min(1, 'Name is required')
  - `email`: z.string().email('Please enter a valid email')
  - `message`: z.string().min(10, 'Message must be at least 10 characters')
  - `website`: z.string().max(0, 'Bot detected').optional().default('') (honeypot)
- Handler: check honeypot first (silently return success if filled), then send email via Resend
- Use `onboarding@resend.dev` as the "from" address for now (real domain not configured until Phase 4). Add a comment: `// TODO Phase 4: Change to hello@zerofluff.co.uk after domain verification`
- Send to `andy@zerofluff.co.uk`, set replyTo to the submitter's email
- Subject: `New enquiry from ${input.name}`
- HTML body: formatted with name, email, and message
- On Resend error: throw ActionError with code 'INTERNAL_SERVER_ERROR'
- Return `{ success: true }` on success

**Step 2: Create src/components/ContactForm.astro** -- Follow the research code example closely:
- Form with `id="contact-form"`, class `contact-form stack`, `novalidate` attribute
- Fields: name (text, required), email (email, required), message (textarea rows=5, required)
- Each field: label + input + span.field-error with id `{field}-error` and `aria-live="polite"`
- Honeypot: div with `class="visually-hidden" aria-hidden="true"`, input with `tabindex="-1"` and `autocomplete="off"`
- Submit button: `button.btn` type="submit" "Send Message"
- Status div: `id="form-status"` with `aria-live="polite"`
- Scoped styles: form-group flex column layout, input/textarea styling using design tokens (--space-sm padding, --color-border, focus ring with accent color), field-error red at --text-sm, .visually-hidden with absolute positioning off-screen
- Client script (not is:inline): import `actions` and `isInputError` from `astro:actions`. On submit: preventDefault, clearErrors, disable button + show "Sending...", call `actions.contact(formData)`. On error: if isInputError show field-level errors, else show generic message. On success: reset form, show "Thanks! Your message has been sent. I'll be in touch soon." with success class.
- Bind to both `DOMContentLoaded` and `astro:page-load` for ClientRouter compatibility
